# UI

## UI Type

Web browser — server-rendered HTML via Jinja2 templates. Minimal vanilla CSS for usability. No JavaScript framework in v0.1.

## Views / Screens

### Screen: Home

**Route:** `GET /`

**Purpose:** Primary entry point showing both first-level entities — Data Sources and Sessions — side by side. User can manage data sources and create/open/delete sessions from this one screen.

**Layout:** Two-column layout (stacks on mobile).

**Left column — Data Sources:**
- Section title: "Data Sources"
- "Upload CSV" inline form (file input + button)
- List of DataSource cards, each showing:
  - Name (filename), type badge, row count, column count
  - Column name chips (first 5)
  - Delete button with inline confirmation ("Delete? Yes / No")
- Empty state if no data sources yet

**Right column — Sessions:**
- Section title: "Sessions"
- "New Session" button → opens inline form below the button showing:
  - Session name field (optional)
  - Checkbox list of all available data sources (must pick ≥ 1)
  - "Create Session" submit button
  - "Cancel" to close the form
- List of Session cards, each showing:
  - Session name and creation date
  - Data source chips (names of all attached data sources)
  - Query count
  - "Open" button → `GET /sessions/{id}`
  - Delete button with inline confirmation
- Empty state if no sessions yet

**Actions:**
- Upload CSV → `POST /datasources/upload`
- Delete DataSource → `POST /datasources/{id}/delete`
- Create Session → `POST /sessions` (with selected data_source_ids)
- Open Session → `GET /sessions/{id}`
- Delete Session → `POST /sessions/{id}/delete`

---

### Screen: Session (Chat)

**Route:** `GET /sessions/{session_id}`

**Purpose:** The main interaction screen. Shows all attached data sources, a question form, and all past Q&A for this session inline.

**Key elements:**
- Breadcrumb: Home › Session name
- Session header: session name + chips for each attached data source
- "Ask a question" card (always on top):
  - Text input
  - Submit button
  - Loading overlay while pipeline runs
- All past Q&A cards inline (newest first), each showing:
  - Question, answer, SQL iterations, token usage, cost
  - Collapsible agent reasoning trace
  - Highlighted with indigo border if it is the `?new=` record
- Stub-mode banner

**Actions:**
- Submit question → `POST /sessions/{session_id}/query`

---

### Screen: Error

**Route:** rendered inline on 4xx/5xx

**Key elements:** Error message + "Back" link.

---

## Error States

- Pipeline errors: render `error.html` — never bare 500
- File upload errors: flash message on home
- 404: render `error.html` with "not found" message

## Tech Stack

Jinja2 + FastAPI. Minimal inline CSS. No JS frameworks. Stub-mode banner via `llm_provider` context variable.
